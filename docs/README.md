# WorshipFlow Documentation

This folder contains durable project documentation.

- `product/` - product scope, domain notes, and feature intent.
- `design/` - design system notes and UI guidance.
- `deployment/` - deployment and production operations notes.
- `development/` - local development workflows.
- `specs/` - version-controlled feature and implementation specs.
- `archive/` - retired or superseded documentation.

Keep root files such as `README.md`, `AGENTS.md`, `DESIGN.md`, and `tokens.css`
in place unless the application conventions change.
